# Track: Coursera & Udemy Subtitle Handler Fixes

## Overview

Comprehensive fix and improvement of the Coursera and Udemy subtitle handlers based on deep analysis. The analysis identified 12+ issues across both handlers plus 4 cross-cutting coordinator issues, ranging from silent functional failures (missed CDN subtitles, lost tracks due to locale collapsing) to quality improvements (dead code, missing tests, silent error swallowing).

## Background

The Coursera handler (`inject/subtitleHandlers/coursera.ts`) and Udemy handler (`inject/subtitleHandlers/udemy.ts`) implement the `SubtitleHandler` interface for URL pattern matching, metadata extraction, and subtitle transformation. Both were originally built in Phase 2 (2026-04-09) and have received incremental fixes (sprite filtering, deep-analysis hardening) but retain structural issues in domain detection, CDN coverage, metadata parsing, and test coverage.

## Functional Requirements

### FR-1: Coursera CDN Subtitle Interception (P1)
Coursera serves VTT subtitles from CloudFront CDNs (e.g. `d3c33hcgiwev3.cloudfront.net`). The handler must include CDN URL patterns so these subtitles are intercepted and translated. The background allowlist already includes `cloudfront.net`.

### FR-2: Coursera videoId Extraction Fix (P1)
The `extractAvailableTracks` method extracts `videoId` via `/lecture/([^/?]+)/` from the metadata API URL, but metadata URLs are `/api/onDemandLectureVideos...` which don't contain `/lecture/`. Fix extraction to pull videoId from the API response body or a different URL segment.

### FR-3: Udemy Locale Region Preservation (P1)
The `extractAvailableTracks` method collapses `locale_id` (e.g. `en_US`) to just `en` via `split('_')[0]`, causing `en_US` and `en_GB` tracks to collide in the coordinator's dedup. Preserve the full locale as the language identifier (replace `_` with `-` per BCP-47).

### FR-4: Udemy Sprite Filter False Positive Fix (P1)
The cue-level sprite filter regex `/^(.+)\.(jpg|png|...)(#xywh=.+)?$/i` makes `#xywh=` optional, so short legitimate subtitles ending in image extensions (e.g. "See photo.png") are wrongly dropped. Require `#xywh=` to be present, or add a no-spaces constraint, since sprite filenames never contain spaces.

### FR-5: Domain-Anchored detect() (P2)
Both handlers use `hostname.includes('coursera.org')` / `hostname.includes('udemy.com')` which matches spoofed subdomains like `notcoursera.org`. Standardize to exact hostname or `.endsWith()` matching, consistent with YouTube's `/\.youtube\.com$/` pattern.

### FR-6: Coursera subtitlesVtt Array Handling (P2)
`extractAvailableTracks` treats `subtitlesVtt` identically to `subtitles` (assumed `{lang: url}` map), but `subtitlesVtt` can be an array of `{languageCode, url}` objects. Normalize based on `Array.isArray`.

### FR-7: Coursera Variable Shadowing Fix (P2)
The `for (const [lang, url] of Object.entries(subtitles))` loop shadows the outer `url` parameter. Rename to `subtitleUrl`.

### FR-8: Udemy data.results Fallback Removal (P2)
The `data?.asset?.captions || data?.captions || data?.results` fallback chain includes `data?.results`, which doesn't match Udemy's lecture API response structure and could match unrelated arrays. Remove the `data?.results` fallback.

### FR-9: Udemy isAutoGenerated Source Detection (P2)
`c.source === 'auto'` only catches one variant. Expand to handle `'auto'` and `'auto-generated'` values.

### FR-10: Use parseSubtitles for Format Auto-Detection (P3)
Both handlers' `transformResponse` calls `parseWebVTT` only. Switch to `parseSubtitles()` from `lib/subtitleParser.ts` which auto-detects VTT vs SRT, adding SRT support for free.

### FR-11: Udemy Dead Code Removal (P3)
Remove the redundant `if (filteredCues.length === 0 && cues.length > 0) return [];` branch — returning `filteredCues` (which is `[]`) is identical.

### FR-12: Error Logging in extractAvailableTracks (P3)
Both handlers silently swallow errors with `catch { return []; }`. Add `console.warn` logging for debuggability when API response structures change.

### FR-13: Udemy Language Extractor Improvement (P2)
The language extractor returns the first 2-letter path segment, which could be a lecture ID. Improve to match Udemy's actual URL structure and handle underscore/3-letter variants.

### FR-14: Coursera Language Extractor Improvement (P3)
The regex `/\/([a-z]{2}(-[A-Z]{2})?)\//` only matches 2-letter codes. Extend to handle 3-letter codes and underscore variants.

### FR-15: Remove Duplicated isWatchPage Fallbacks (Cross-cutting)
`content/subtitleCoordinator.ts` `isOnWatchPage()` has hardcoded fallbacks for `udemy.com` and `coursera.org` that duplicate the handler `isWatchPage()` methods. Remove the duplicated fallbacks since the coordinator already prefers `handler.isWatchPage()` when available.

### FR-16: Full Unit Test Coverage
Add comprehensive unit tests for both handlers covering: `detect()`, `isWatchPage()`, `getMetadataPatterns()`, `extractAvailableTracks()` (with realistic API response fixtures), and language extractors. These methods currently have zero test coverage despite containing the bugs being fixed.
