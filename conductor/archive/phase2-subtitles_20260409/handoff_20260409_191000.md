## Key Implementation Decisions

1. **MAIN world injection via WXT `defineContentScript`** — Uses `world: 'MAIN'` and `runAt: 'document_start'` in `entrypoints/inject.content/index.ts`. No `web_accessible_resources` manual config needed; WXT handles it.
2. **PostMessage bridge channel `lingua-lens`** — Simple string channel filter with `requestId` correlation. No origin validation beyond channel check (both worlds are same-origin).
3. **Interceptor pattern** — XHR and Fetch interceptors store match metadata on the request object (`__linguaLensMatch`) to avoid double-matching in `open` and `send`.
4. **Handler registry pattern** — Handlers implement `SubtitleHandler` interface (`detect`, `getPatterns`, `transformResponse`). Registry is a module-level array, not a class — simpler for the MAIN world context.
5. **YouTube dual format** — srv3 XML parsed via DOMParser, JSON3 via JSON.parse. Format detected from URL `fmt` param or content sniffing.

## Learnings from This Section

### Patterns Discovered
- WXT `defineContentScript` with `world: 'MAIN'` auto-configures `web_accessible_resources` — no manual manifest editing needed.
- ESLint strict config (`@typescript-eslint/strict`) forbids `no-this-alias` — use captured local variables from `this` (e.g., `const registry = this.registry`) instead of `const self = this`.
- ESLint strict also forbids `prefer-rest-params` over `arguments` — must use `...args` in patched prototypes.
- `@eslint/js` 10.x has unmet peer dep with eslint 9.x — works but warns.

### Gotchas Encountered
- Message bridge tests: must properly track `addEventListener`/`removeEventListener` calls to verify cleanup works. Using a Map with string keys fails because function references serialize identically — use an array with identity comparison instead.
- `no-dupe-else-if` lint error: when `if` and `else-if` conditions overlap (first condition is a superset), the else-if is unreachable.
